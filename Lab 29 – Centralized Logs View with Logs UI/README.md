# Lab 29 – Centralized Logs View with Logs UI

## 📌 Lab Overview

In this lab, we implemented a centralized logging system using **Filebeat** and **Kibana Logs UI**.

The purpose of this lab was to understand how logs from multiple servers can be collected, forwarded, centralized, and analyzed using the Elastic Stack.

Filebeat was configured as a lightweight log shipper to collect system and application logs and send them to Elasticsearch, where they can be visualized and filtered through Kibana Logs UI.

# Task 1: Enable Filebeat on Multiple Servers

## Step 1.1: Install Filebeat

Filebeat is an open-source lightweight data shipper used to forward log data.

Install Filebeat:

```bash
sudo apt-get update
sudo apt-get install filebeat -y
```

---

## Step 1.2: Configure Filebeat

Edit Filebeat configuration:

```bash
sudo nano /etc/filebeat/filebeat.yml
```

Configure log input:

```yaml
filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/*.log
```

Configure Elasticsearch output:

```yaml
output.elasticsearch:
  hosts: ["localhost:9200"]
```

---

## Step 1.3: Start Filebeat Service

Start Filebeat:

```bash
sudo systemctl start filebeat
```

Enable Filebeat at system boot:

```bash
sudo systemctl enable filebeat
```

Check status:

```bash
sudo systemctl status filebeat
```

# Task 3: Filter Logs by Host or Service

## Filter Logs by Host

Example query:

```
host.name: "server1.example.com"
```

This displays logs generated by a specific host.

---

## Filter Logs by Service

Example:

```
service.name: "apache"
```

This filters logs related to Apache service.

---

## Advanced Filtering

Combine multiple conditions:

```
host.name: "server1.example.com" AND service.name: "apache"
```

Kibana Query Language (KQL) allows analysts to perform advanced searches on log data.
